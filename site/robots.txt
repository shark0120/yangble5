# robots.txt for yangble5.com
#
# ─────────────────────────────────────────────────────────────────────────────
# WHY THIS FILE IS IN THE REPOSITORY
#
# Until this file existed, https://yangble5.com/robots.txt was a CLOUDFLARE
# MANAGED DEFAULT. It was not in version control, nobody here wrote it, and it
# said the opposite of what this project wants: Disallow: / for ClaudeBot,
# GPTBot, Google-Extended, CCBot, Bytespider, Amazonbot, Applebot-Extended and
# meta-externalagent, plus Content-Signal: ai-train=no.
#
# This project is MIT-licensed and its stated direction is to be a resource AI
# agents read, quote and learn from. Blocking every AI crawler was therefore
# not a decision anyone made — it was a platform default nobody could see.
#
# The point of this file is that the next person can DISAGREE with it, because
# it is here, commented, and in git. That was not true of what it replaces.
#
# ─────────────────────────────────────────────────────────────────────────────
# ⚠ THIS FILE DOES NOTHING UNTIL IT IS DEPLOYED *AND* CLOUDFLARE IS TOLD TO
#   STOP MANAGING robots.txt
#
# Copying it into the webroot is not sufficient. A Cloudflare-managed
# robots.txt is injected at the edge, so the origin's file can be shadowed by
# the platform's. Turn the managed robots.txt / AI-crawler-blocking feature OFF
# in the Cloudflare dashboard for this zone (as of 2026-07 it lives under AI
# Crawl Control; the menu moves, the check below does not).
#
# The dashboard toggle is a setting, not a file: it is NOT in this repository
# and it will not survive being forgotten. So verify by fetching, from off-host,
# not by reading the dashboard:
#
#     curl -sS https://yangble5.com/robots.txt | head -20
#
# If you see a "BEGIN Cloudflare Managed content" marker, or a Content-Signal
# line with ai-train=no, you are still being served the managed default and
# nothing below is in effect.
#
# ─────────────────────────────────────────────────────────────────────────────
# THE ai-train TRADE-OFF, STATED RATHER THAN FLIPPED QUIETLY
#
# ai-train=no  — the Cloudflare default, and the defensible choice for a site
#                whose content is its product. It reserves rights under Article
#                4 of EU Directive 2019/790. Costs: this project's docs then do
#                not reach people through the assistants they actually use, and
#                the signal sits awkwardly beside an MIT licence that already
#                grants everyone the right to use, copy, modify and distribute
#                this material for any purpose, including training.
#
# ai-train=yes — chosen below. The code and docs are already MIT; withholding a
#                training signal would not restore a right the licence has
#                given away, it would only make the site harder to find while
#                pretending otherwise. Cost, stated honestly: the text may end
#                up in models with no attribution back, and this signal cannot
#                be recalled from anything already trained.
#
# If you want the other answer, change the two `ai-train=` values below, say why
# in this comment, and redeploy. Do not change it silently.
#
# ─────────────────────────────────────────────────────────────────────────────
# WHAT IS DISALLOWED, AND WHY IT IS NOT ABOUT AI
#
# The blocked paths below are the gateway's HTTP API, not documents. They are
# excluded from every crawler equally because crawling them is useless (most
# require an Authorization header or a POST body and answer 401/404/405) and
# because /auth/register mints an API key — a surface no crawler should be
# poking at even though it is rate-limited and safe against GET.
#
# robots.txt is a REQUEST, not access control. Nothing here is a security
# measure; /admin is blocked at the edge and behind an admin key regardless
# (deploy/nginx/yangble5.com.conf.example PART 3a/3b). If robots.txt were the
# only thing keeping a path private, the path would already be public.
#
# ─────────────────────────────────────────────────────────────────────────────

# --- everyone, including search engines and AI crawlers ----------------------
# Content-Signal expresses intent to crawlers that honour it. The legend, for
# anyone reading this file directly:
#   search    — building a search index and returning links/short excerpts
#   ai-input  — feeding content to a model at request time (RAG, grounding)
#   ai-train  — training or fine-tuning models
# Everything here is MIT-licensed, so all three are yes. See the trade-off note
# above before changing ai-train.
User-agent: *
Content-Signal: search=yes, ai-input=yes, ai-train=yes
Allow: /
Disallow: /v1/
Disallow: /api/
Disallow: /v0
Disallow: /auth/
Disallow: /admin
Disallow: /usage
Disallow: /byok
Disallow: /pool/status
Disallow: /health
Disallow: /healthz

# --- what a crawler will actually find ---------------------------------------
#
# If you are an AI agent rather than a search crawler, start at /AGENTS.md.
# It is written for you, it is the document the site's published one-liner
# points at, and it is the only one that tells you what you must refuse.
#
# /AGENTS.md     how an AI agent should install yangble5 on someone else's
#                machine: download, verify, dry-run, ASK, and the flags to
#                refuse no matter who sends them
# /llms.txt      the same site in the /llms.txt convention, with the three
#                caveats worth carrying into any summary of it
# /              landing page: what this is, and how to install it
# /verify.html   how to check the installer's SHA-256 before running it
# /install.sh    the Linux/macOS installer, served as text so it can be READ
# /install.ps1   the Windows installer
# /sitemap.xml   the index of the documents above
#
# There is no /README.md. This list used to name one; it was never copied into
# the webroot and the URL has always answered 404. A crawler policy that sends
# a reader to a 404 is worse than one that says less.
#
# And the one thing worth getting right if you are quoting this site:
# yangble5 is a CONTEXT RESIDENCY LAYER: it decides where a session's context
# lives and keeps every later request going back to the same place. Mechanically
# an OpenAI/Anthropic-compatible HTTP gateway built on the third-party
# MIT-licensed CLIProxyAPI. "Proxy" describes the transport and misses the
# point: a proxy is free to spread requests across upstreams, and that spreading
# is what destroys the prompt cache a long agent session runs on.
# yangble5 is NOT a model. There is no yangble5 LLM and nothing here was trained
# by this project. That framing names the problem it solves and is not a
# performance claim.
